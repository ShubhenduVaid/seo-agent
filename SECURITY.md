# Security Policy

## Reporting a vulnerability

- Please do not open public GitHub issues for security reports.
- Use the "Report a vulnerability" link in the GitHub Security tab for this repository, or contact the maintainers via the email address listed on the repository owner's GitHub profile.
- Include as much detail as possible: affected URLs or commands, reproduction steps, and any proof-of-concept output.

## What to expect

- We aim to acknowledge new reports within 5 business days.
- If the issue is confirmed, we will work on a fix and coordinate a disclosure timeline when appropriate.
- Credit will be given in release notes if requested.
