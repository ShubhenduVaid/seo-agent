# Discoverability Checklist

These steps improve how the project appears in GitHub search and to newcomers.

## GitHub repository settings (manual)

1. **Topics**: add relevant topics such as:
   - technical-seo
   - seo-audit
   - site-audit
   - seo-tool
   - crawler
   - cli
   - python
2. **Description**: keep the short description specific and benefit-led.
3. **Homepage**: point to the README or docs entry page.
4. **Social preview**: upload a social card (1200x630) to improve link sharing.
5. **Discussions**: enable to capture questions and ideas from users.

## Repository hygiene

- Keep `README.md` concise and updated.
- Maintain `CHANGELOG.md` for releases.
- Use consistent labels (bug, enhancement, docs, good first issue).

## Releases

- Tag releases with `vX.Y.Z` and add release notes.
- Consider a short release summary to help users decide to upgrade.

## Community

- Keep issue templates and PR templates up to date.
- Add a few `good first issue` tickets to encourage contributors.
